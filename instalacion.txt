# Instalación GPSWOX to SUTRAN Retransmitter - Rocky Linux 9.4

## Repositorio
https://github.com/ArmandoCH85/apiwoxsutran

---

## 1. Requisitos del sistema
- Rocky Linux 9.4 (Blue Onyx)
- RAM: 3.6 GiB (disponible ~1.8 GiB)
- El servicio consume ~50MB RAM

---

## 2. Actualizar sistema
sudo dnf update -y

## 3. Instalar Node.js 20
sudo dnf install -y nodejs npm
node --version

## 4. Crear usuario para el servicio
sudo useradd -m -s /bin/false gpsretransmitter

## 5. Clonar repositorio desde GitHub
cd /home
sudo git clone https://github.com/ArmandoCH85/apiwoxsutran.git gpsretransmitter
sudo chown -R gpsretransmitter:gpsretransmitter /home/gpsretransmitter

## 6. Instalar dependencias
cd /home/gpsretransmitter
npm install --production

## 7. Configurar variables de entorno
sudo nano /etc/environment

Agregar estas líneas:
GPSWOX_API_HASH=tu_hash_aqui
SUTRAN_ACCESS_TOKEN=tu_token_aqui

Guardar y salir (Ctrl+X, luego S, luego Enter).

## 8. Configurar el servicio

Crear archivo de servicio:
sudo nano /etc/systemd/system/gps-retransmitter.service

Copiar y pegar este contenido:

[Unit]
Description=GPSWOX to SUTRAN Retransmission Service
After=network.target

[Service]
Type=simple
User=gpsretransmitter
WorkingDirectory=/home/gpsretransmitter
EnvironmentFile=/etc/environment
ExecStart=/usr/bin/node /home/gpsretransmitter/src/index.js
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

Guardar y salir.

## 9. Habilitar y arrancar servicio
sudo systemctl daemon-reload
sudo systemctl enable gps-retransmitter
sudo systemctl start gps-retransmitter

## 10. Verificar funcionamiento

Ver estado del servicio:
sudo systemctl status gps-retransmitter

Ver logs en tiempo real:
sudo journalctl -u gps-retransmitter -f

---

## OBTENER CREDENCIALES

### GPSWOX API Hash
Obtener un nuevo hash haciendo login:
curl -X POST https://drsecuritygps.com/api/login -d "email=sutran@drsecurity.com&password=Sutran2026"

La respuesta incluye user_api_hash - usar ese valor en GPSWOX_API_HASH.

### SUTRAN Access Token
Usar el token proporcionado: e712932b-8558-4d50-b1e0-6911e117a0b8

---

## COMANDOS ÚTILES

Reiniciar servicio:
sudo systemctl restart gps-retransmitter

Ver todos los logs:
sudo journalctl -u gps-retransmitter --no-pager

Detener servicio:
sudo systemctl stop gps-retransmitter

Deshabilitar inicio automático:
sudo systemctl disable gps-retransmitter

---

## CONFIGURACIÓN

Editar config.yaml para cambiar intervalo de polling:
cd /home/gpsretransmitter
nano config.yaml

Ejemplo - cambiar a 60 segundos:
gpswox:
  polling_interval_ms: 60000

Después de cambiar, reiniciar:
sudo systemctl restart gps-retransmitter

---

## SOLUCIÓN DE PROBLEMAS

Si el servicio no inicia:
sudo journalctl -u gps-retransmitter -n 50

Ver variables de entorno cargadas:
sudo systemctl show-environment

Ver último error:
sudo systemctl log -u gps-retransmitter --priority=err

---

## ACTUALIZAR DESDE GITHUB

Para actualizar a la última versión:
cd /home/gpsretransmitter
sudo git pull origin main
sudo systemctl restart gps-retransmitter

---

## NOTAS

- El servicio envía datos cada 30 segundos (configurable)
- Solo transmite vehículos con IMEI válido de 15 dígitos
- Vehículos con datos >20 días no se transmiten
- Los 7 MB de RAM son más que suficientes para este servicio
